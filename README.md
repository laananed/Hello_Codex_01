# 潮の花静态页面

## 项目简介

这是一个小型静态网站项目，入口文件是 `index.html`，样式文件是 `styles.css`，视觉素材放在 `assets/` 目录下。当前页面是一个日文二次元插画风格的落地页，围绕海风、花瓣、柔和光影和角色氛围组织内容。

项目目前没有 `package.json`、构建脚本、JavaScript 入口文件或自动化测试配置，因此不需要安装前端依赖，也没有打包流程。

## 功能特点

- 静态 HTML + CSS 实现，无前端框架依赖。
- 单页结构，包含顶部导航、主视觉区、说明区、特色区和场景区。
- 使用 `assets/hero-reference.png` 作为主视觉图片。
- 页面内锚点导航指向 `#home`、`#story`、`#features` 和 `#gallery` 等区块。
- CSS 使用渐变背景、玻璃拟态卡片、浮动装饰元素和响应式布局样式。

## 安装步骤

当前项目不需要安装依赖。

1. 克隆或下载项目到本地。
2. 进入项目目录：

   ```bash
   cd Hello_Codex_01
   ```

3. 确认项目文件存在：

   ```bash
   git status --short
   ```

> 说明：仓库中当前没有 `package.json`，因此不要运行 `npm install`、`pnpm install` 或 `yarn install`，除非后续新增了包管理配置文件。

## 启动命令

当前没有配置开发服务器或启动脚本。可以直接用浏览器打开入口文件：

```bash
open index.html
```

如果当前系统不支持 `open` 命令，也可以在文件管理器中双击 `index.html`，或使用浏览器的“打开文件”功能选择该文件。

也可以临时使用本地静态服务器预览，例如在安装了 Python 的环境中运行：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

该命令只是本地预览方式，不是项目内置脚本。

## 文件结构

```text
Hello_Codex_01/
├── AGENTS.md
├── README.md
├── assets/
│   └── hero-reference.png
├── index.html
└── styles.css
```

主要文件和文件夹作用：

- `index.html`：页面入口文件，包含页面结构、文案、导航锚点和图片引用。
- `styles.css`：页面样式文件，负责布局、颜色、字体、背景、卡片、按钮、动画和响应式表现。
- `assets/`：静态资源目录，当前包含主视觉图片 `hero-reference.png`。
- `AGENTS.md`：仓库协作和代码风格说明文件，不属于页面运行所需文件。
- `README.md`：项目说明文档。

## 常见问题

### 项目需要安装依赖吗？

不需要。当前仓库没有 `package.json` 或其他包管理配置文件。

### 为什么没有 `npm run dev` 或 `npm start`？

因为当前项目是简单静态页面，没有配置 Node.js 开发服务器、构建工具或启动脚本。

### 页面入口文件在哪里？

入口文件是项目根目录下的 `index.html`。

### 图片资源放在哪里？

图片资源放在 `assets/` 目录下。当前页面引用了 `assets/hero-reference.png`。

### 修改后如何验证？

可以直接在浏览器打开 `index.html`，检查桌面和移动端布局、图片加载、锚点跳转以及文字在背景上的可读性。

### 是否包含 API Key、密码、token 或私钥？

本项目当前是静态页面。从现有项目结构看，没有需要读取或展示 API Key、密码、token 或私钥的配置文件。维护文档或代码时也不要提交此类敏感信息。
